# Verify Web/Node.js SDK

The Verify SDK (`@succinctlabs/zcam1-verify`) validates the authenticity of C2PA-signed photos and videos in the browser and Node.js.

## Installation

```bash
npm i @succinctlabs/zcam1-verify
```

## Core Concepts

The Verify Web/Node.js SDK:
- Extracts C2PA manifests from photos and videos
- Computes the file content hash
- Validates bindings (Apple App Attest signatures) for files with `succinct.bindings`
- Validates ZK proofs against a pinned Apple Root CA for files with `succinct.proof`
- Extracts capture metadata (device info, camera settings, etc.)
- Confirms the file was captured on a genuine iOS device

## API Reference

### Classes

- [VerifiableFile](./VerifiableFile) — Main class for verifying C2PA-signed files in the browser
- [VerifiableBuffer](./VerifiableBuffer) — Main class for verifying C2PA-signed files in Node.js

### Methods

- [authenticityStatus](./authenticityStatus) — Determine the authenticity status of a file
- [verifyBindings](./verifyBindings) — Verify the hardware-backed Apple App Attest signature
- [verifyProof](./verifyProof) — Verify the zero-knowledge proof
- [captureMetadata](./captureMetadata) — Extract capture metadata (device info, camera settings)
- [c2paReader](./c2paReader) — Access the underlying C2PA reader
- [dataHash](./dataHash) — Compute the content hash of a file

## Complete Example

```ts
import { VerifiableFile, AuthenticityStatus } from "@succinctlabs/zcam1-verify";

const input = document.querySelector<HTMLInputElement>("#file-input")!;

input.addEventListener("change", async () => {
  const file = input.files?.[0];
  if (!file) return;

  const verifiable = new VerifiableFile(file);

  const statusResult = await verifiable.authenticityStatus();

  if (statusResult.isErr()) {
    console.error("No valid manifest found");
    return;
  }

  const status = statusResult.value;

  if (status === AuthenticityStatus.Bindings) {
    const bindingsResult = await verifiable.verifyBindings(true);
    console.log("Bindings valid:", bindingsResult.isOk() && bindingsResult.value);
  } else if (status === AuthenticityStatus.Proof) {
    const proofResult = await verifiable.verifyProof("TEAM_ID.com.example.app");
    console.log("Proof valid:", proofResult.isOk() && proofResult.value);
  } else {
    console.log("Invalid manifest");
  }

  const metadataResult = await verifiable.captureMetadata();
  if (metadataResult.isOk()) {
    console.log("Captured at:", metadataResult.value.when);
    console.log("Parameters:", metadataResult.value.parameters);
  }
});
```

## What Verification Checks

| Check | What It Validates | Use Case |
|-------|-------------------|----------|
| `verifyBindings()` | Apple App Attest signature is valid and corresponds to the file hash | For files with `succinct.bindings` (before proof generation) |
| `verifyProof()` | ZK proof is valid and chains to the Apple Root CA | For files with `succinct.proof` (after proof generation) |

**Verification Flow:**

For files with `succinct.bindings`:
1. Call `verifyBindings()` to verify the Apple App Attest signature

For files with `succinct.proof`:
1. Call `verifyProof()` to verify the ZK proof

## Notes

- All verification methods use `ResultAsync` from `neverthrow` for error handling
- Use `verifyBindings()` for files with `succinct.bindings` (before proof generation)
- Use `verifyProof()` for files with `succinct.proof` (after proof generation)
- Files with only `succinct.bindings` will fail `verifyProof()` (they haven't been proven yet)
- The Apple Root CA is pinned in the SDK; proofs must chain to this root
- `captureMetadata()` provides access to device and camera settings recorded at capture time
