# Editing a ZCAM Photo

All zPhotos captured with ZCAM are C2PA-compatible, so any C2PA-aware editor can preserve the original capture manifest while adding an edit manifest.

Common editing tools today include Capture One, Adobe Lightroom, and PhotoLab. Capture One and Adobe Lightroom support C2PA exports; PhotoLab does not. Typically editors include the content credentials on export.

## Exporting with Content Credentials

When exporting a zPhoto from Capture One, include the C2PA content credentials in the export settings. This preserves the original capture manifest and adds an edit manifest describing the changes.

![Exporting with content credentials in Capture One](/images/screenshot_captureone_c2pa.png)

After export, the image will contain a C2PA manifest that includes both the original capture information and the edit history.

![C2PA manifest showing capture and edit history](/images/screenshot_manifest_captureone.png)

## Verification Limitations After Editing

Both bindings verification and ZK proof verification will fail if a zPhoto is edited. This happens because:

- The Apple App Attest signature (used by bindings verification) covers the hash of the original image
- The ZK proof (used by proof verification) also attests to the original image hash
- Editing changes the image bytes and therefore the hash
- As a result, signature and proof verification fail when the hashes no longer match

**Important**: Although C2PA edit manifests can preserve the original capture and add edit history, the cryptographic signature/proof tied to the original image will not verify against the edited bytes.

If you must edit a zPhoto after capture, you need to generate a new signature or proof after editing; otherwise verification will fail.

### Future Work: ZK Proofs for Edit Chains

In principle, zero-knowledge proofs could be extended to verify edit chains while preserving privacy. A ZK proof could assert that:

1. The original zPhoto was captured authentically
2. Each edit in the chain was made or authorized by trusted editors
3. The final image hash corresponds to the verified edit chain

This approach would require:

- Proof generation after each edit (incurs computation cost)
- Including edit history in the proof circuit
- More complex SP1 programs to verify multiple C2PA manifests and their relationships

For now, users who need edits should either generate a new signature/proof after editing or accept that verification of the edited image will fail. The current implementation focuses on verifying the original capture, which provides the strongest guarantees for unedited zPhotos.
